# 食材养生公众号文章生成工具 - 使用教程

## 一、准备工作

### 1. 解压文件
将下载的压缩包解压到电脑任意位置（建议放在桌面，方便找到）。

### 2. 安装 Node.js
这是运行本工具的必要环境。

**安装步骤：**
1. 打开浏览器，访问 https://nodejs.org/
2. 点击绿色的 **"LTS"** 按钮下载（推荐稳定版）
3. 下载完成后双击安装，一路点击 **"下一步"** 即可
4. 安装完成后，按 `Win + R` 键，输入 `cmd` 回车
5. 在黑色窗口中输入：`node -v`
6. 如果显示版本号（如 v18.x.x），说明安装成功

> 💡 如果显示"不是内部或外部命令"，请重启电脑后再试

---

## 二、配置 API 密钥

本工具使用 **ModelScope（魔塔）** 平台，一个密钥即可同时生成文字和图片。

### 申请 ModelScope 访问令牌

1. 访问 https://modelscope.cn/ 注册账号（支持手机号/邮箱/支付宝快捷登录）
2. 点击右上角头像 → "我的" → "访问令牌"（Access Token）
3. 点击"新建访问令牌"，复制生成的 Token
4. 用记事本打开 `config.json` 文件，将 Token 填入：

```json
{
  "apiUrl": "https://api-inference.modelscope.cn/v1/chat/completions",
  "apiKey": "你的ModelScope访问令牌",
  "model": "Qwen/Qwen2.5-72B-Instruct",
  "imageApiKey": "你的ModelScope访问令牌"
}
```

> 💡 **提示**：`apiKey` 和 `imageApiKey` 可以填同一个令牌，ModelScope 每天提供 **10000 Token 免费额度**，足够生成多篇图文。

### 可选：使用其他 API（高级用户）

如需使用其他平台（如讯飞星火、DeepSeek），只需修改：
- `apiUrl`：API 地址
- `apiKey`：你的密钥
- `model`：模型名称（如 `deepseek-chat`）
- `imageApiKey`：保持不变或删除（图片仍需 ModelScope）

## 三、运行工具

### 方法一：双击运行（最简单）

找到 `双击运行.bat` 文件，双击即可启动。

### 方法二：命令行运行

1. 按 `Win + R`，输入 `cmd` 回车
2. 输入 `cd `（注意 cd 后面有个空格）
3. 将解压后的文件夹拖到命令窗口，自动填入路径
4. 按回车
5. 输入 `node run.js` 回车

---

## 四、使用流程

### 第 1 步：生成选题
运行程序后，会自动生成 6 个养生选题，例如：
```
📝 今日选题：

  1. 睡前吃3颗这个，血管干净人轻松
  2. 这5种食物是"天然消炎药"，厨房里就有
  ...
```

### 第 2 步：选择选题
输入序号 1-6 选择想要的选题，回车确认。

### 第 3 步：等待生成
程序会自动：
- ✍️ 生成完整文章
- 🖼️ 生成配套图片（封面、食材图、场景图）
- 📄 合成公众号 HTML 文件

等待过程中请不要关闭窗口。

### 第 4 步：获取成果
生成完成后，打开 `outputs` 文件夹，里面有：
- `YYYY-MM-DD/` 文件夹（按日期命名）
  - `选题池.md` - 今日所有选题
  - `文章标题.html` - 公众号文章（含图片）
  - `cover_封面.jpg` - 封面图
  - `food_食材名.jpg` - 食材配图
  - `scene_场景.jpg` - 场景配图

## 五、导入公众号

### 方法一：直接复制粘贴（推荐）

1. 用浏览器打开生成的 `.html` 文件
2. 全选内容（Ctrl + A），复制（Ctrl + C）
3. 登录公众号后台 → 新建图文消息
4. 粘贴到正文，调整格式即可

### 方法二：使用编辑器导入

1. 安装 135 编辑器 / 秀米编辑器 / 壹伴插件
2. 将 HTML 内容复制到编辑器中
3. 微调排版后同步到公众号

## 六、常见问题

**Q1：运行报错 "Cannot find module"？**
> 在项目文件夹内运行 `npm install`，等待安装完成后再运行。

**Q2：提示 API 错误？**
> 检查 `config.json` 中的密钥是否正确：
> - 是否已注册 ModelScope 账号并创建访问令牌
> - Token 是否复制完整（不要有多余空格）
> - 账号是否有足够额度（ModelScope 每天 10000 Token 免费额度）

**Q3：图片生成失败？**
> 检查以下几点：
> 1. `config.json` 中的 `imageApiKey` 是否正确（ModelScope 访问令牌）
> 2. 是否已安装 Python 3（在命令行输入 `python --version` 检查）
> 3. 首次运行时需要安装 Python 依赖，检查网络连接是否正常
>
> 如果提示缺少 Python 库，手动运行：`pip install requests Pillow`

**Q4：生成的文章不满意？**
> 可以重新运行选择不同选题，或修改 `prompts/article.md` 调整生成风格。
